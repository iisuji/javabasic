[ 상속 ]
	-> 클래스끼리 부모자식 관계  

1. 개발자 관점( 자식클래스 입장 )
	- 부모의 멤버를 사용 
	- 부모에 원하는 멤버가 없으면 추가해서 사용 가능 
	- 부모 멤버 변경해서 사용 가능 (*) ... overriding(오버라이딩)
	
	[참고] 부모 메소드를 똑같이 기술 
	
	[참고] overloading과 overriding 차이점 
     overriding 메소드 : 부모랑 나랑 똑같은 메소드를 가지고 있는 것 
     - 부모 클래스에 있는 걸 변경해서 쓰고 싶을 때, 자녀 클래스 안에서 똑같은 메소드를 만들어 오버라이딩 한 다음에 거기에서 수정하면 됨.
     - 오버라이딩 했을 때, 자녀 클래스 우선으로 적용되기 때문에! 
     
    [참고] 자바의 모든 클래스 Object 클래스를 상속받는 구조 
    	  단일 상속 ( 부모는 무조건 하나만 와야 함 --> extends Umma, Appa (x) )

     
2. 설계자 관점 
	- 공통적인 요소를 부모 클래스에 지정
	- 고유 기능은 각자의 자식 클래스에 지정
